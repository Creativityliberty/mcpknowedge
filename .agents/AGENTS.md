# Workflow Nümtema ProjectFlow

> [!IMPORTANT]
> L'agent DOIT AUTOMATIQUEMENT tracer son travail dans Nümtema ProjectFlow de manière proactive. À chaque fois que vous terminez une implémentation significative, un refactoring ou avant de faire un `git push` :
> 1. Vous devez documenter les nouvelles tâches complétées dans la mémoire ProjectFlow.
> 2. Vous devez exécuter la vérification finale.
> 3. Vous devez toujours lancer l'outil `projectflow.handoff` ou `projectflow.resume` pour sceller les traces.
> 4. Vous ne devez pas attendre que l'utilisateur vous le demande. Faites-le de manière autonome et transparente avant de clôturer votre session de travail.
